Automating SSDT build and deployment with Jenkins from a local git repo

2017-04-04

This is a short illustration of using a local installation of Jenkins on Windows to build an SSDT project from a local git repo and deploy it to a SQL Server on the same machine.

This is probably useful for a quick demonstration or to understand how the various moving parts fit together, but possibly less applicable to “Real Life” production environments.

There are no build agents and no git remotes; all the action takes place on the Jenkins master, and the git repo is local to the same machine. No “git management” software such as GitHub/VSTS/GitLab/BitBucket/etc is involved (except for some slight cheating regarding the build definition itself.)

Getting set up

I am using Windows 10 with SQL Server 2016 Developer Edition and Visual Studio 2015 Professional. Most of this stuff should work with most editions and versions of SQL Server and Visual Studio, but some of the paths will vary here and there. I am using SQL Server Data Tools Version 14.0.61021.0, released on October 26, 2016.

Installing Jenkins

I used the windows installer linked here (note that this link will trigger the download immediately). At the time of writing this is version 2.46.1. In the initial setup, I selected the option to “Install Recommended Plugins”; this installs more than is required for this example.

Further Jenkins twiddling

The MSBuild plugin needs to be installed separately (as it isn’t “recommended”!), which can be done from http://localhost:8080/pluginManager/available or by clicking through Jenkins -> Manage Jenkins -> Manage Plugins -> Available. We need to tell Jenkins where it can find MSBuild.exe, this is done on http://localhost:8080/configureTools/ (Jenkins -> Manage Jenkins -> Global Tool Configuration). You can find out where MSBuild is installled by opening the “Developer Command Prompt for Visual Studio” and typing

C:\Program Files (x86)\Microsoft Visual Studio 14.0>where msbuild
C:\Program Files (x86)\MSBuild\14.0\Bin\MSBuild.exe
C:\Windows\Microsoft.NET\Framework\v4.0.30319\MSBuild.exe

I used the first of these, as this is the one installed by Visual Studio1 2015, and should avoid having to re-read the somewhat convoluted StackOverflow thread here.

I already had git installed and in the path, so there was no need to configure this separately in Jenkins.

$ git --version
git version 2.11.0.windows.3

Compromising your SQL Server 2

By default, the Jenkins service will run as “Local System” on Windows. In order to allow this account - which will authenticate to SQL Server as the machine account - to deploy the database, I made “Local System” an sa of the SQL Server:

EXEC sp_addsrvrolemember 'NT AUTHORITY\SYSTEM', 'sysadmin';

There are many circumstances in which this wouldn’t be a good idea, if any of them apply in your scenario you should make appropriate adjustments to the scripts that follow.

Creating the Jenkins job

The first step is to create the Jenkins job that will build our project into a dacpac, and deploy it to a local SQL Server.

The job definition is contained in the following Jenkinsfile, which consists of three fairly self-explanatory stages. The first stage git checkout checks out our master branch from the local (indicated by the file:\\ prefix) repo. The second stage calls the MSBuild tool we defined earlier, taking advantage of the fact that our project is very simple to provide very few parameters on the command line. The third calls out to the shell to call sqlpackage.exe, again with very few parameters. This last stage is the one that requires whatever account Jenkins is running under to be able to authenticate to the SQL Server (though there are alternatives involving storing credentials in Jenkins).

Conveniently, since this is stored in a GitHub gist, and a gist is a git repo, I can create a new pipeline job and specify “Pipeline Script from SCM” in the build definition then provide the url of the gist for the repository url.

Note that this repo contains a single file, namely the build definition - all the project files are coming from the git repo on our local machine. If you want to copy and paste the complete url, it’s in the build output log reproduced below.

One more detail is that the job needs to be set to “Poll SCM”, but the schedule can be left empty (ignore the warning). This isn’t required to test the build, but will be required later on to trigger the build on a commit to our git repo.

Triggering the build from the git repo

Many approaches to building a project automatically on each commit rely on the CI server, in this case Jenkins, polling the source control system at some pre-defined interval.

However, if the geo-political events of 2016 are any guide, polling doesn’t always lead to the outcome we expect, so in this case we’ll create a git “hook” to trigger the build on every commit to master.

A git hook is a script that runs in response to a specific event that occurs in git. There are a bunch of events that can fire hooks, these can be used to enforce commit message conventions, run static code analysis, prevent force-pushes, and all kinds of other arbitrary busywork. The event that’s of interest here is the post-commit hook, which runs after a commit has been added. In “Real Life”, hooks are normally added to the server copy of the repo but since there’s no remote in our setup, everything has to be local.

Notably, even though we’re on Windows in this example, hooks are executed by the msysgit subsystem so much of the usual UNIX shell stuff will work.

Hooks are stored in the .git folder in our repo (this is usually hidden on Windows) in files that match the name of the hook. So, our file is called post-commit (no extension) and contains the #!/bin/sh “shebang” plus a single line that uses curl to hit a specific URL in the Jenkins server that will cause our Jenkins job to poll its source repo as defined in the build pipeline (i.e. file:///C:/Projects/Chinook.JenkinsDemo )

C:\Projects\Chinook.JenkinsDemo\.git\hooks>type post-commit
#!/bin/sh
curl http://localhost:8080/git/notifyCommit?url=file:///C:/Projects/Chinook.JenkinsDemo

Note for git beginners: this file isn’t in the repo when you download it, you’ll need to add it yourself!

Now, all that remains is to make an arbitrary change to our project and add a commit.

If everything is working the pipeline job should be triggered, and the new build will also log that it was “Started by an SCM Change”.

To clarify what’s happened here, we made a change and committed it to our local repo, the post-commit hook called the url in Jenkins, and Jenkins then “polled” the git repo, found a new commit, and triggered the pipeline job.

Until next time, happy triggering!

Как попасть в свой размер

Важно понимать, что часто продавцы в размерной таблице указывают замеры вещи, а не параметры фигуры. Поэтому при выборе посмотрите, есть ли фото на модели и указали ли параметры модели (многие хорошие магазины так делают для того, чтобы было проще выбрать).

Я обычно ориентируюсь на отзывы других покупателей, ищу параметры, схожие с моими и смотрю, как это сидит на человеке. Путем опыта пришла к выводу, что на мой рост 170, ОГ 93, ОТ 70 и ОБ 95 лучше всего садится L (реже XL), а на размер ноги в 24,6 - 9 размер обуви.

Общие рекомендации: Сомневаешься? Бери на размер больше. С мужскими вещами – на два размера, так как у китайцев есть странная любовь к тому, чтобы вещи сидели в обтяжку, что очень не нравится русским мужчинам.

Как сэкономить 1. Через свою корзину с товарами подписываться на продавцов и получать от них купоны. Есть бестолковые «аттракционы невиданной щедрости» типа «потрать 10 тысяч и получи скидку 65 рублей», а есть очень и очень симпатичные. 2. Применять спецкупоны на распродаже. Это относительно недавнее обновление на AliExpress, их можно потратить на определённые товары в определенных магазинах (увы, не всех), и можно совмещать с другими скидками и купонами. Как пользоваться купонами? Найдите товары, которые вы хотите оплатить спецкупонами. При оформлении заказа из корзины перейдите на страницу оплаты, и сумма к оплате автоматически уменьшится на номинал купона. 3. Кроме того, накануне распродажи 11.11 очень рекомендую поиграть в «Шопоголию» в мобильном приложении Это игра, в которой Вы сможете получить ещё больше купонов, а самое главное - получить часть от 1.000.000$. Пройдите до конца, займет 10 минут всего вашего времени. 4. Через приложение участвовать в конкурсах AliExpress – их всегда много, в них есть возможность выиграть каждому (достаточно сделать пост по теме конкурса с красивыми фото и связным текстом), и купоны раздаются очень выгодные – от 20 до 100 долларов при любой покупке. Такие конкурсы находятся во вкладке Интересное/Вдохновение и обычно указаны в баннерах.

Как не дать себя обмануть

Использовать приложения, отслеживающие динамику цен – это помогает увидеть, действительно ли продавец снизил цену на распродаже, или еще больше накрутил стоимость. Не вестись на заманчиво дешевые предложения типа куртки со скидкой 99% - так продавцы накручивают себе количество заказов перед распродажами. Смотреть отзывы с реальными фотографиями, не стесняться писать в разделе «Мои вопросы» и уточнять информацию у тех, кто уже купил вещь.

Уровень покупателя Уровень покупателя рассчитывается из нескольких параметров – чем он выше, тем больше вы получаете от сайта приятных бонусов (свои собственные купоны, дополнительные скидки) и защиты в случае споров. Его можно посмотреть во вкладке Мой профиль. Он обновляется раз в месяц 8 числа, и зависит от того, сколько вы потратили за предыдущий месяц, покупали ли товары редких категорий, как много отзывов на товары написали и отвечали ли на вопросы других покупателей (поэтому очень рекомендую это делать!). Кроме того, точно так же, как мы ставим баллы полученному товару, так и продавцы ставят баллы нам как покупателям. Споры и возвраты не влияют на рейтинг, а вот необоснованная отмена заказа – да. Поэтому не ведитесь на просьбы продавцов отменить заказ (бывают и такие, кто пишет вам, что товар закончился, и просят вас это сделать). Так вот в случае отмены заказа продавцом минус идет ему, в случае же, если это сделали вы, да еще и не указали причину – заминусят вас.

Споры Я в основном открывала споры по причине того, что товар так и не дошел (и чаще всего это происходило по вине Почты России) – часто товар отслеживается вплоть до территории РФ, а потом исчезает. В таких случаях я дожидаюсь окончания срока защиты покупателя (он отслеживается на странице Мои заказы) и нажимаю на Открыть спор. При неполучении заказа даже фото/скринов не требуется, система и так все видит. Указываешь сумму, которую заплатил, и ждешь одобрения спора и возврата средств. Важный нюанс – купоны, которые были использованы при покупке такой вещи, не возвращаются. Я так потеряла крутой купон на 30 долларов, когда продавец просто не отправил мне товар… было очень обидно. В нескольких случаях я открывала спор по причине брака – сломанная вещь, сколы, замятия – и тогда, конечно же, к форме открытия спора прилагала фотографии. Кстати их необходимо сжимать перед прикреплением – стандартное качество фотографий оказывается слишком большое для платформы.

Единственный мой проигранный спор был связан с несоответствием описания товара и реальности, но продавец поменял описание на главной, и я осталась ни с чем. С тех пор всегда делаю скрины страницы до открытия спора.

Преимущества площадки AliExpress

Невероятное разнообразие необычных вещей, которые у нас могут продаваться в специализированном магазине с накруткой в 5-10 раз. Уникальные корейские и китайские бренды одежды – да, они чуть дороже в среднем по площадке, но их качество и дизайн того стоят Копии брендов, которые по качеству пошива/материала идентичны оригиналам (в Китае расположены заводы по производству многих брендов, так что неудивительно). Зара, Манго и инстаграмные шоурумы все представлены на AliExpress (а многие инста-магазины выдают китайские вещи за свои и перепродают с гигантской наценкой). Многие европейские бренды одежды приходят на AliExpress – в частности, теперь там есть Vero Moda и IceBear. Оперативность обновления ассортимента и отслеживание модных тенденций – я несколько раз покупала на али вещи из Зары, которые только спустя пару месяцев появлялись в оффлайн магазинах. А сумочка-пельмешек а ля Боттега вообще появилась у меня чуть ли не первой в Москве. Постоянный рост качества товара и обслуживания клиентов – да, есть магазины-однодневки, но большинство магазинов понимает выгоду в долгосрочном сотрудничестве и старается поддерживать уровень. Так, один магазин бесплатно выслал мне блузку взамен другой, у которой разошелся шов.

Недостатки AliExpress

Большое количество дешевого шлака и вещей, рассчитанных на дурной вкус. Увы, площадка чутко реагирует на запросы пользователей и подстраивается под них. Долгое ожидание Иногда часть стоимости товара прячется в платной доставке. Это делается для того, чтобы при поиске товар выглядел дешевле, чем он есть Либо же, при бесплатной доставке, магазин может начать просить тебя оплатить доставку якобы для большей безопасности товара. Делать это, конечно же, не следует (выше я писала, почему). Накрутка заказов некоторыми магазинами, а также обман покупателя – неверные фото товара, замеры, вводящие в заблуждение и тп. Навязчивые сообщения об акциях от магазинов, в которых что-то покупал прежде (но их можно отключить).

И все же при всех недостатках сайта – он растет и развивается, и это здорово, как и общая тенденция к повышению качества. Сайт рекомендую от всего сердца! Если есть какие-то вопросы, которые я не осветила, пишите, отвечу.
